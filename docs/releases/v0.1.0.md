# Tile 0.1.0

Tile's first public release provides a compact Python runtime for headless,
tool-using agent sessions.

## Highlights

- Task-owned runs that continue independently of event subscribers.
- Persistent sessions with in-memory and SQLite conversation history stores.
- Structured streaming events and replayable, provider-neutral conversation
  records.
- Pydantic-defined tool inputs, runtime validation, and model-correctable tool
  failures.
- Built-in `read`, `bash`, `edit`, `grep`, `find`, `ls`, and `write` tools.
- Typed prompt outcomes through caller-supplied Pydantic result models.
- Structured run failure diagnostics with original in-process exceptions
  retained for local debugging.
- OpenAI Responses API support through a caller-constructed `AsyncOpenAI`
  client.

## Install

```bash
pip install tile-runtime==0.1.0
```

The distribution is named `tile-runtime`; the Python import is `tile`.

## Current scope

Tile 0.1.0 supports Python 3.13 or newer and the OpenAI Responses API. Its APIs
are intentionally 0.x and may change without deprecation cycles. Durable run
records, cross-process event replay, approval resumption, additional providers,
and service mode are planned rather than included in this release.

The built-in tools are unconfined and execute with the permissions of the Tile
process. Use operating-system isolation when tool execution requires a security
boundary.
